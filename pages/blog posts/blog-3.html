Performance and optimization - January 24, 2020.

Validation

Any web company that is about to release their website to the public, always make sure to follow a validation process as it helps to eliminate any errors that a coder can’t notice from their naked eye. “validating web document is really important, as it helps improve ensuring the quality which saves a lot of time” (W3.org, 2013). So, this is a necessary step that everyone should be making as using a debugging tool will allow the user to get notified if mistakes have been made, therefore, this improves the efficiency of the standard code. There are two things that I validated which were the HTML and the CSS which have a different validator for each one. For validating the HTML I usedW3C Markup Validation Service and to validate my CSS I used W3C CSS Validation Service.

Optimisation

Speed is declared to be a crucial facet when creating a website. There are tons of websites on the internet today, which advocates the users to spend less time on one particular website. If the website isn’t loading quickly, they are likely to be on the next website in a moment of few seconds, therefore, it’s vital for a website to have any media content to be compressed to the maximum level; eradicating any areas of concern related to speed. So, the first action I took to make my website more efficient was to optimize all the media content, which is also known as image optimization. In my case I had to optimize all of the animated gifs, the reason being that, gifs tend to become much bigger in size when compiled together, which causes the file size to increase. I used an online compressor called ezgif, to reduce the file size. The other images were saved in the png format and optimized by using tinypng.
